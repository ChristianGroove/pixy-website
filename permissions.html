1. Resumen de la Aplicación

Pixy es una plataforma SaaS (Software as a Service) de gestión empresarial diseñada para pequeñas y medianas empresas. Permite a las empresas:

  • Gestionar sus contactos y clientes (CRM)
  • Centralizar comunicaciones de múltiples canales en un inbox unificado
  • Generar cotizaciones y facturas
  • Automatizar flujos de trabajo
  • Ofrecer un portal de autoservicio a sus clientes

Información de la Aplicación:

  • Nombre: Pixy
  • Tipo: Business / B2B SaaS
  • URL: app.pixy.com.co
  • Empresa: Pixy SAS (Colombia)

2. Caso de Uso Principal

Los usuarios de Pixy son agencias de marketing, consultoras y empresas de servicios que gestionan comunicaciones con sus clientes a través de múltiples canales.

Ejemplo de flujo:

  1. Un cliente envía un mensaje de Instagram Direct a la página de la agencia
  2. El mensaje aparece en el inbox unificado de Pixy
  3. El equipo de la agencia responde desde Pixy
  4. La respuesta se envía automáticamente a Instagram
  5. El contacto se asocia al CRM para seguimiento

Beneficio: Las empresas no necesitan revisar múltiples aplicaciones; todo está centralizado en Pixy.

3. Permisos de Facebook

3.1 pages_messaging

Permiso pages_messaging
¿Qué permite? Enviar y recibir mensajes de Facebook Messenger en nombre de Páginas administradas por el usuario
¿Por qué lo necesitamos? Para mostrar mensajes de Messenger en el inbox unificado y permitir responder desde Pixy
Datos accedidos Mensajes entrantes/salientes, nombre del remitente, ID de conversación
Cómo se usan Se muestran en el inbox, se asocian a contactos del CRM, se almacenan para historial

3.2 pages_manage_metadata

Permiso pages_manage_metadata
¿Qué permite? Suscribirse a webhooks de la Página para recibir notificaciones en tiempo real
¿Por qué lo necesitamos? Para recibir notificaciones instantáneas cuando llega un nuevo mensaje

3.3 pages_read_engagement

Permiso pages_read_engagement
¿Qué permite? Leer comentarios y reacciones en publicaciones de la Página
¿Por qué lo necesitamos? Para incluir comentarios en el inbox y responder a interacciones

4. Permisos de Instagram

4.1 instagram_basic

Permiso instagram_basic
¿Qué permite? Acceder a información básica del perfil de Instagram Business/Creator
Datos accedidos ID de cuenta, nombre de usuario, foto de perfil

4.2 instagram_manage_messages

Permiso instagram_manage_messages
¿Qué permite? Enviar y recibir mensajes de Instagram Direct
¿Por qué lo necesitamos? Para centralizar DMs de Instagram en el inbox unificado

4.3 instagram_manage_comments

Permiso instagram_manage_comments
¿Qué permite? Leer y responder comentarios en publicaciones de Instagram
¿Por qué lo necesitamos? Para gestionar comentarios desde el mismo inbox

5. Permisos de WhatsApp Business

5.1 whatsapp_business_messaging

Permiso whatsapp_business_messaging
¿Qué permite? Enviar y recibir mensajes de WhatsApp Business API
¿Por qué lo necesitamos? Para integrar WhatsApp Business en el inbox unificado
Datos accedidos Mensajes, números de teléfono de contactos, estado de entrega

5.2 whatsapp_business_management

Permiso whatsapp_business_management
¿Qué permite? Gestionar la cuenta de WhatsApp Business (plantillas, configuración)
¿Por qué lo necesitamos? Para verificar el estado de la cuenta y gestionar plantillas de mensajes

6. Flujo de Datos

A continuación se muestra cómo fluyen los datos desde las plataformas de Meta hacia Pixy:

┌─────────────────┐     Webhook      ┌─────────────────┐
│   Facebook      │ ───────────────► │                 │
│   Messenger     │                  │                 │
└─────────────────┘                  │                 │
                                     │                 │
┌─────────────────┐     Webhook      │     PIXY        │
│   Instagram     │ ───────────────► │                 │
│   Direct        │                  │    (Inbox)      │
└─────────────────┘                  │                 │
                                     │                 │
┌─────────────────┐     Webhook      │                 │
│   WhatsApp      │ ───────────────► │                 │
│   Business      │                  │                 │
└─────────────────┘                  └────────┬────────┘
                                              │
                                              ▼
                                     ┌─────────────────┐
                                     │   Base de Datos │
                                     │   (Encriptada)  │
                                     └─────────────────┘

Proceso:

  1. Usuario conecta su cuenta de Meta con Pixy
  2. Pixy se suscribe a webhooks para las páginas/cuentas autorizadas
  3. Cuando llega un mensaje, Meta envía un webhook a Pixy
  4. Pixy almacena el mensaje en la base de datos encriptada
  5. El mensaje aparece en el inbox del usuario
  6. El usuario responde desde Pixy
  7. Pixy envía la respuesta a través de la API de Meta

7. Medidas de Seguridad

  • Encriptación en tránsito: TLS 1.3 para todas las comunicaciones
  • Encriptación en reposo: AES-256 para datos almacenados
  • Tokens de acceso: Almacenados de forma segura, nunca expuestos al cliente
  • Row Level Security (RLS): Aislamiento de datos entre organizaciones
  • Auditoría: Logs de acceso a datos sensibles
  • Eliminación de datos: Proceso automatizado para solicitudes de eliminación
